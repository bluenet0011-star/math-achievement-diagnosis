# 수학 성취기준 진단 웹 — Phase 1 (확률과 통계)

2022 개정 교육과정 기반, 학생이 활동형 문항에 답하며 개념 위계(계통도)를 따라 결손을 진단하는 단일 페이지 웹.
설계 근거는 [제작 계획서.md](제작%20계획서.md) 참고.

## 실행 방법
JSON 콘텐츠를 `fetch`로 불러오므로 **로컬 서버**가 필요하다 (파일 더블클릭은 브라우저 보안정책에 막힘).

```bash
cd "성취 평가 사이트"
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

GitHub Pages에 올리면 그대로 동작한다.

## 진단 흐름 (현재 구현 — 일반/정밀)
1. **시작** — 학년 · 진단 방식(일반/정밀) · 단원 선택
2. **계통도** — 과목→단원→성취기준 위계. 성취기준 클릭 시 우측에 세부 수행능력 표시 → 진단 시작
3. **활동** — 최상위 성취기준부터 과정(step)별 문항. 막히거나 "모르겠어요" 선택 시 **선수개념으로 역추적**. 상단바에 **진단 진행률 막대** 표시. (종합 진단형 — 정답은 마지막에 공개)
4. **리포트** — 전체 도달률(링), 성취기준별 결과 카드(**도달도 진행바** + "모르겠어요" 표시), 보강 추천, 계통도 색칠. localStorage 이어하기 지원.

- **일반진단**: 선택한 단원의 최상위 성취기준만 점검 → 막히면 역추적
- **정밀진단**: 과목 전체 단원의 최상위 성취기준 점검

## 계통도 (graphview.js)
중·고 전 과목(16과목·314 성취기준)을 하나의 거대 계통도로 표시한다. Cytoscape.js + compound 노드(과목▸단원▸성취기준 묶음) + preset 배치로 **겹침 없이** 그린다.
- **학년/단계별 세로 배열** — 중학교(위) → 고1 공통/기본 → 고2 일반선택 → 진로/융합선택(아래)으로 단계가 위에서 아래로 쌓이고, 선수개념은 위쪽·선수관계선은 아래로 흐른다. 왼쪽에 단계 라벨 표시.
- **시각 디자인**(보고서 [계통도_시각디자인_개선보고서.md](계통도_시각디자인_개선보고서.md) 반영) — 학년 띠 배경, 줌아웃 시 라벨 자동 숨김(min-zoomed-font-size), 최상위(진한 테두리)·★(금색) 강조, 간선 위계(검수=실선/초안=점선/직접추가=초록)와 범례, 진단 후 미평가 노드는 흐리게 처리해 결과 강조, "노드 색 = 과목/도달도" 모드 표기.
- **직교(round-taxi) 연결선**(보고서 [계통도_연결선_개선보고서.md](계통도_연결선_개선보고서.md) 반영) — 선이 박스 사이 세로 통로를 따라 직각으로 흐르며(위→아래) 박스 위에 그려져 가려지지 않음. 조직도/가계도식 정렬로 곡선 엉킴(헤어볼) 완화. 완전 노드회피가 필요하면 ELK 자동 라우팅을 향후 옵션으로.
- **선 기본 숨김** — 성취기준을 클릭하면 그 개념의 **선행(선수) 성취수준 체인만** 강조되고 나머지는 흐려짐
- **「전체 선 보기」 토글** — 모든 선수관계선을 한 번에 표시
- **확대·축소·이동** — 휠 줌, 드래그 팬, +/−/전체보기 버튼, 과목 점프 드롭다운
- **빠른 선행 탐색** — 우측 패널의 선행 성취수준을 클릭하면 해당 노드로 이동·강조
- **검색** — 성취기준 코드/문구 검색 → 해당 노드로 이동·강조(Enter로 다음 결과)
- **도달도 필터** — 진단 후 완전/부분/미흡/미평가별로 노드 표시·숨김
- **색 분리** — 노드 채움 = 과목(학년)색, **좌측 띠 = 도달도** (색 이중의미 해소)
- **전체 선 보기** 시 선을 **출발 학년색**으로 구분
- **미니맵** — 우측 하단 오버뷰 + 현재 뷰포트 표시·클릭 이동
- **과목 접기/펼치기**(cytoscape-expand-collapse) — 묶음 접어 복잡도 관리
- **자동 정렬(ELK)** — 층 배치·교차 최소화·직교 라우팅 보기로 토글(학년 띠는 잠시 숨김), 다시 누르면 학년 배열 복귀

## 교사 편집 모드 (계통도 우측 상단 「✎ 편집 모드」)
명세서 B3·D2·A2 — 위계·수행능력을 코드 수정 없이 편집. 두 탭으로 구성.

**[선수관계] 탭**
- **추가**: 선수(아래) 개념 → 상위 개념 순으로 두 노드를 클릭 (순환·중복 자동 차단)
- **삭제**: 선을 클릭 → "이 선수관계 삭제"
- **★ 중요 표시**: 선택한 성취기준 토글
- **선수관계 내보내기(JSON)** → `data/edges/` 에 저장하면 전체 반영

**[세부 수행능력] 탭** (명세서 A2 — 과목별 성취수준·하위 수행능력 정리)
- 성취기준 노드를 클릭 → 그 성취기준의 세부 수행능력 목록을 추가/삭제
- 진단 판정은 수행능력별 통과 비율로 산출되므로 핵심 데이터
- **수행능력 내보내기(JSON)** → `data/nodes/` 의 해당 성취기준 `skills` 에 반영

**불러오기(import)** — 내보낸 `prereq-edited.json` / `skills-edited.json` 을 「불러오기」 버튼으로 다시 적용. **여러 파일을 한 번에 선택**하면 순차 병합되어 여러 교사의 편집을 합치거나 백업을 복원할 수 있다.
- 선수관계 파일은 전체 스냅샷이므로 **베이스(파일 원본) 대비 차이로 환산**해 기존 편집과 병합(파일 우선: 파일에 있으면 추가, 베이스에 있는데 파일에 없으면 삭제).
- 수행능력 파일은 성취기준별 목록을 **노드 단위로 덮어써** 병합.
- 적용 후 자동 새로고침으로 반영. 존재하지 않는 노드/형식 오류 항목은 건너뛴다.

모든 변경은 **localStorage 자동 저장**(새로고침 유지), **초기화** 버튼으로 전체 취소.
(백엔드 도입 시 내보내기/불러오기 자리에 저장 API만 연결 — 명세서 E3·F2)

## 폴더 구조
```
index.html        # Cytoscape/dagre CDN + 앱 스크립트
css/styles.css
js/
  data.js        # JSON 로더 + 그래프 헬퍼 (단일 출처)
  graphview.js   # Cytoscape 계통도 (배치·강조·줌·선 토글)
  templates.js   # 활동 유형 템플릿 (render/grade 계약)
  engine.js      # 진단 상태 기계 (모드/역추적/판정/종료)
  app.js         # 화면 흐름 + 부트스트랩
data/
  manifest.json          # 불러올 콘텐츠 목록 (추가 시 여기만 수정)
  nodes/prob-stats.json  # 확률과 통계 (큐레이션: 세부 수행능력·isTop)
  nodes/all-subjects.json# 중·고 전 과목 자동 추출 (확통 제외)
  edges/edges.json         # 큐레이션 간선 (확통 내부 + 교과 간)
  edges/prereq-curated.json# 교과 간/학년 간 선수관계 1차 초안 (위계 기반)
  edges/prereq-draft.json  # 영역 내 순차 자동 초안 (교사 검수 필요)
  activities/*.json       # 성취기준별 활동(문항) — 확통 일부만 시연용
templates/활동작성_프롬프트.md  # 교사용 고정 저작 프롬프트
```

## 데이터 현황
- 노드 387 (성취기준 314 / 16과목), 간선 659, 끊긴 간선 0.
- **선수관계 간선 대부분은 자동 1차 초안**(영역 내 순차 연결)이며 `draft:true`로 표시 — 교사·연구회 검수로 다듬을 대상(명세서 D2).
- 활동(문항)은 채우지 않음 — 교사가 `templates/활동작성_프롬프트.md` 규격으로 제작.

## 콘텐츠 추가 (코드 수정 없이)
- **성취기준/수행능력**: `data/nodes/<과목>.json`
- **선수관계**: `data/edges/edges.json` (`from`=선수개념 → `to`=상위개념)
- **활동(문항)**: `data/activities/<과목>.json` (작성 규격은 `templates/활동작성_프롬프트.md`)
- 새 파일은 `data/manifest.json`에 경로만 등록.

## 데이터 현황 (Phase 1)
- 확률과 통계 성취기준 16개 + 공통수학1 경우의 수 1개(교과 간 연결) = 노드 전부 등록.
- 활동(문항)은 진단 시연용으로 8개 성취기준에 부착(확률 단원 + 선수개념 체인).
- 나머지 성취기준은 활동 추가 시 자동으로 진단 대상에 포함된다.
